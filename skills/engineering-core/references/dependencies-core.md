# Dependency Selection and Integration

## Selection Criteria

- Prefer actively maintained libraries with strong documentation.
- Verify compatibility with the runtime/framework versions in use.
- Prefer well-supported choices over niche alternatives.
- Record why each dependency is needed.

## Integration Protocol

- Read official documentation before implementation.
- Inspect existing project patterns before introducing new integration shapes.
- Document setup, configuration, and uninstall considerations.
- Keep integration behavior observable and reversible.
