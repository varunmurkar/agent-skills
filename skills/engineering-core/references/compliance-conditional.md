# Conditional Compliance Guidance

Apply this guidance only when a specific compliance regime is explicitly required.

## Compliance Mode Gate

Enable only when:
- explicitly requested, or
- project docs/config declare compliance obligations.

If not enabled, do not add compliance-specific process overhead.

## Rules (When Enabled)

- Document compliance-sensitive code paths and controls.
- Keep auditability for critical security and data handling behaviors.
- Align testing and review checklists with declared compliance requirements.
