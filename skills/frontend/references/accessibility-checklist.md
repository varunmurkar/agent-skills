# Accessibility Checklist

- Use semantic HTML first, ARIA second.
- Ensure full keyboard operability for interactive controls.
- Maintain readable contrast and avoid color-only cues.
- Keep heading hierarchy logical.
- Give icon-only controls accessible names.
- Ensure predictable visible focus.
- Manage dialog focus enter/exit behavior.
- Announce important dynamic updates when needed.

## Verification

- Run an accessibility audit for changed surfaces.
- Spot-check complex interactions with a screen reader.
- Validate at least one keyboard-only user path.
